<objective>
Implement exact-revision dataset adapters and the explicit opt-in source-ingestion boundary that stages canonical rows and verified source facts for the later partition/audit transaction.
</objective>

<must_haves>
- GSM8K, ProofWriter, and bAbI source identity, configuration, official splits, row metadata, and licenses are pinned and retained.
- Network access is false by default and possible only through `prepare-data --allow-download`; tests inject rows/resolvers and never call the Hub.
- Preparation rejects unresolved/ambiguous revisions, source schema drift, invalid source gold, unexpected splits/counts, or unsafe source substitution.
- The resulting staged source lock and normalized source shards contain hashes sufficient for the partition/audit transaction to fail closed on mutation; this plan must not mark a prepared corpus `COMPLETE`.
</must_haves>
