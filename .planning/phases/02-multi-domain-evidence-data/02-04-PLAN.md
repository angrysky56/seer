<objective>
Build the offline, resumable Qwen generation-and-scoring corpus, enforce immutable per-regime budgets, report natural class sufficiency without adaptive repair, and obtain explicit operator acceptance of real prepared/model evidence.
</objective>

<must_haves>
- Exact tokenizer chat templating produces one system/user prompt and records prompt/template/token identities; Qwen control tokens are never hand-built.
- Primary non-thinking generation is one greedy bounded prediction per selected example; the hash-selected thinking subset is separate, sampled with recorded seeds, and capped at 1024 new tokens.
- Resume appends only missing generation IDs and rejects changed model/tokenizer, prompt, parameters, data-lock, partition, or audit identity.
- Generation and scoring are separate capabilities: generation consumes prompt/ID-only views and atomically seals immutable shards before gold-bearing scoring begins.
- Sufficiency counts natural scored predictions only and reports `underpowered` at any 99/100 boundary without resampling, retries, regime merging, task expansion, or token-budget changes.
- Real network preparation and cached-Qwen/GPU generation remain explicit manual acceptance steps; offline fake tests are the automated gate.
</must_haves>
