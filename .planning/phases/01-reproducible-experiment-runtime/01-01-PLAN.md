<objective>
Establish a strict, versioned experiment configuration and the complete Phase 1 CLI contract, with a synthetic smoke entry point and explicit placeholders for later-phase commands.
</objective>

<must_haves>
- Invalid, incomplete, unknown-key, and unsupported-version configurations fail with field-qualified diagnostics.
- Canonical serialization and digesting are stable across round trips and exclude derived runtime facts.
- The installed `seer` command exposes prepare-data, cache-outputs, train, evaluate, run-matrix, build-report, and smoke without performing network access during parsing or synthetic dispatch.
- The checked-in synthetic configuration represents all Phase 1 configuration groups and can be loaded without implicit defaults hiding required identity fields.
</must_haves>
